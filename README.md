# Telegram Bot для списка покупок

Совместный список покупок с шаблонами, категориями и автоматической сортировкой по отделам супермаркета.

## Возможности

- Совместный список покупок для семьи/компании
- Автоматическая категоризация продуктов (молочные, мясо, овощи и т.д.)
- Шаблоны для быстрого наполнения списка
- Объединение дубликатов с конвертацией единиц (кг+г, л+мл)
- Управление доступом: новые пользователи проходят одобрение админа
- Еженедельное напоминание в субботу в 19:00

## Локальный запуск

```bash
# Установить uv (если нет)
curl -LsSf https://astral.sh/uv/install.sh | sh

# Установить зависимости
uv sync

# Создать .env из примера
cp .env.example .env
# Заполнить BOT_TOKEN и ADMIN_ID

# Запустить
uv run python bot.py
```

## Запуск через Docker

### Требования

- Docker и Docker Compose
- VPS: 1 ядро CPU, 512 МБ RAM, 1 ГБ диск

### Быстрый старт

```bash
# Клонировать репозиторий
git clone <repo-url> grocery_shopping && cd grocery_shopping

# Создать .env
cp .env.example .env
nano .env  # Заполнить BOT_TOKEN и ADMIN_ID

# Запустить
docker compose up -d
```

### Полезные команды

```bash
docker compose logs -f          # Логи контейнера
docker compose restart          # Перезапуск
docker compose down             # Остановить
docker compose up -d --build    # Пересобрать и запустить
```

### Папки

| Папка | Назначение |
|-------|-----------|
| `data/` | SQLite база данных (сохраняется между перезапусками) |
| `logs/` | Логи с ротацией: `bot.log` до 5 МБ, хранится 3 копии |

## Конфигурация

Создайте бота через [@BotFather](https://t.me/BotFather) и узнайте свой Telegram ID через [@userinfobot](https://t.me/userinfobot).

Переменные окружения (`.env`):

| Переменная | Обязательна | Описание | По умолчанию |
|-----------|-------------|----------|-------------|
| `BOT_TOKEN` | Да | Токен Telegram бота | — |
| `ADMIN_ID` | Да | Telegram ID администратора | — |
| `DATABASE_PATH` | Нет | Путь к файлу БД | `/app/data/grocery.db` |
| `LOG_DIR` | Нет | Папка для логов | `logs` |

## Команды

### Для всех пользователей

| Команда | Описание |
|---------|----------|
| `/start` | Регистрация / инструкция |
| `/help` | Справка |
| `/add <продукт>` | Добавить продукт |
| `/list` | Показать список |
| `/done <N>` | Отметить купленным |
| `/undo <N>` | Вернуть в список |
| `/remove <N>` | Удалить |
| `/clear` | Удалить все купленные |

### Для администратора

| Команда | Описание |
|---------|----------|
| `/pending` | Список ожидающих подтверждения |
| `/users` | Список пользователей с доступом |

## Система доступа

1. Новый пользователь пишет `/start` → попадает в список ожидающих
2. Пользователь видит сообщение об ожидании подтверждения
3. Админ использует `/pending` и нажимает **Одобрить** или **Отклонить**
4. Одобренный пользователь получает уведомление

## Формат добавления продуктов

**Интерактивный режим (через кнопку ➕ Добавить):**

1. Введите название продукта
2. Выберите единицу измерения (кг, г, шт, л, мл, уп) или "Без меры"
3. Введите количество

**Быстрый режим (через команду):**

```
/add молоко 2л
/add яйца 10 шт
/add сыр 500г
/add хлеб
```

**Объединение дубликатов** — одинаковый продукт с той же группой единиц суммируется:

```
/add молоко 2л  →  "молоко 2л"
/add молоко 3л  →  "молоко 5л" (обновлено)
/add молоко 1шт →  "молоко 1шт" (новый продукт, другая мера)
```

## Категории

Продукты автоматически распределяются по отделам супермаркета:

Молочные продукты, Мясо, Рыба, Овощи, Фрукты, Хлеб, Бакалея, Замороженные, Консервы, Напитки, Сладости, Бытовое, Другое

Распознавание по словарю ключевых слов + нечёткий поиск. Пользователь может вручную сменить категорию — выбор сохраняется и переиспользуется.

## Шаблоны

- Создание шаблонов вручную или из текущего списка
- Добавление шаблона в список с обработкой конфликтов
- Управление позициями шаблона (добавление, удаление, редактирование)
